## 1. Schema (Drizzle TypeScript) — FINALIZED

```typescript
import {
  pgTable,
  pgEnum,
  uuid,
  varchar,
  boolean,
  timestamp,
  text,
} from "drizzle-orm/pg-core";

// --- Enums ---

export const roleEnum = pgEnum("role", ["TA", "FA", "CA", "CF", "AP"]);

export const authTokenTypeEnum = pgEnum("auth_token_type", [
  "magic_link",
  "password_reset",
]);

// --- Users ---
// Auth-relevant fields only.
// password_hash: nullable; present only when has_password = true (Q1 — Option A confirmed).
// modules: text array; valid module identifiers TBD (Phase 7 / Foundation Doc) (Q2 confirmed).

export const usersTable = pgTable("users", {
  id:            uuid("id").defaultRandom().primaryKey(),
  email:         varchar("email", { length: 255 }).unique().notNull(),
  role:          roleEnum("role").notNull(),
  modules:       text("modules").array().notNull().default([]),       // identifiers TBD — Phase 7
  hasPassword:   boolean("has_password").notNull().default(false),
  passwordHash:  varchar("password_hash", { length: 255 }),           // nullable; set when password opted in
  isActive:      boolean("is_active").notNull().default(true),
  keepLoggedIn:  boolean("keep_logged_in").notNull().default(false),
  lastLoginAt:   timestamp("last_login_at"),
  createdAt:     timestamp("created_at").notNull().defaultNow(),
  updatedAt:     timestamp("updated_at").notNull().defaultNow(),
});

// --- Auth Tokens ---
// Unified table for magic_link and password_reset (D-AUTH-001).
// Raw token is a cryptographically random string; stored value is bcrypt hash.
// Single-use enforced via used_at (NULL = unused).
// Expiry enforced via expires_at (creation + 30 min, per D-ARCH-047).

export const authTokensTable = pgTable("auth_tokens", {
  id:         uuid("id").defaultRandom().primaryKey(),
  userId:     uuid("user_id").notNull().references(() => usersTable.id),
  token:      varchar("token", { length: 512 }).unique().notNull(),   // bcrypt hash of raw token
  type:       authTokenTypeEnum("type").notNull(),
  expiresAt:  timestamp("expires_at").notNull(),                      // created_at + 30 min
  usedAt:     timestamp("used_at"),                                   // NULL = unused; set on consumption
  createdAt:  timestamp("created_at").notNull().defaultNow(),
});

// --- Sessions ---
// Cookie-based, server-side session store (D-ARCH-019 — NOT JWT).
// Session ID (UUID) stored in httpOnly + secure + sameSite=strict cookie named `wdi_session`.
// expires_at = creation time + role-based timeout (D-ARCH-017):
//   TA/FA/CA → 12 hours; CF/AP → 48 hours.
// keep_logged_in (D-ARCH-018): when true at login, applies the role-based timeout as-is
//   (this field's effect is already encoded in expires_at at creation — no runtime extension).
// Fixed expiry confirmed (Q3 — Option A): expires_at never changes after session creation.
// last_active_at: updated on each authenticated request — for audit only, no expiry effect.

export const sessionsTable = pgTable("sessions", {
  id:           uuid("id").defaultRandom().primaryKey(),              // value stored in cookie
  userId:       uuid("user_id").notNull().references(() => usersTable.id),
  expiresAt:    timestamp("expires_at").notNull(),                    // fixed at creation; never updated
  createdAt:    timestamp("created_at").notNull().defaultNow(),
  lastActiveAt: timestamp("last_active_at").notNull().defaultNow(),   // audit only — does not extend expiry
});
```

## 3. Component Plan

### Staff Auth

| Component | Description |
|-----------|-------------|
| `StaffLoginPage` | Email input + "Send magic link" CTA. "Sign in with password" link shown conditionally only if `has_password = true` (D-AUTH-004, D-ARCH-058). |
| `MagicLinkSentScreen` | Confirmation that link was sent. Displays raw magic link URL on-screen with copy-to-clipboard button (D-ARCH-057). "Resend link" action triggers `/api/auth/magic-link/resend`. |
| `PasswordLoginForm` | Password input + submit. Only reachable if `has_password = true`. "Back to magic link" escape link always visible. |
| `ForgotPasswordPage` | Email input to trigger a `password_reset` token. On submit, leads to `MagicLinkSentScreen` (reset variant). |
| `PasswordResetForm` | New password + confirm fields. Validates token via `/api/auth/password/verify-token` before rendering. Covers all 3 entry points: forgot-password email, account screen, TA/FA-initiated email (Q5). |

### Applicant Auth (AP)

| Component | Description |
|-----------|-------------|
| `ApplicantLoginPage` | Entry at `/apply` — email input + "Send magic link". No password option ever (AP = magic link only, always). |
| `ApplicantMagicLinkSentScreen` | Same pattern as `MagicLinkSentScreen` but scoped to AP context and `/apply` routing. |

### Shared / Infrastructure

| Component | Description |
|-----------|-------------|
| `AuthGuard` | Server middleware — validates `wdi_session` cookie against `sessions` table, checks `expires_at`, enforces `is_active`. Redirects unauthenticated requests to role-appropriate login page. |
| `SessionExpiryWarning` | Client-side component — warns user N minutes before `expires_at`. Display-only; no session refresh (fixed expiry confirmed, Q3). Prompts user to log in again before expiry if needed. |
